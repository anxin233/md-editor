## 1. 项目概览

这是一个基于 Electron + Vue 3 的桌面端 Markdown 编辑器，目标体验接近 Typora，支持多标签、源码编辑、所见即所得、分栏预览、文件夹浏览、最近文件、主题切换、自定义 CSS、图片粘贴、Mermaid、公式、代码高亮，以及 PDF / HTML / Word 导出。

主要技术栈：

- 桌面壳：Electron 33
- 前端框架：Vue 3 + Pinia
- 构建工具：Vite 6 + vite-plugin-electron
- 编辑器：
  - 源码模式：CodeMirror 6
  - 所见即所得：Milkdown 7
- Markdown 渲染：markdown-it + GFM/emoji/footnote/front-matter/texmath
- 代码高亮：Shiki
- 图表：Mermaid
- 导出：Electron printToPDF、HTML 写入、html-to-docx

已验证：

- `npm run type-check` 通过。
- IDE 当前未报告 linter 错误。

未验证：

- 未运行 `npm run build`，因为脚本会继续调用 `electron-builder` 生成安装包。
- 项目当前没有测试脚本和测试文件。

## 2. 核心结构理解

### 2.1 Electron 主进程

关键文件：

- `electron/main/index.ts`
- `electron/main/ipc/file.ipc.ts`
- `electron/main/ipc/dialog.ipc.ts`
- `electron/main/ipc/export.ipc.ts`
- `electron/main/ipc/recent.ipc.ts`
- `electron/preload/index.ts`

主进程负责创建无边框窗口，注册 IPC，处理窗口控制、文件读写、文件夹读取、保存图片、文件 watcher、导出、最近文件和系统对话框。

当前安全基础做得不错的部分：

- `nodeIntegration: false`
- `contextIsolation: true`
- `sandbox: true`
- preload 只暴露封装后的 `window.electronAPI`
- 文件访问有 `allowedPaths` 白名单机制

但白名单边界仍有风险，详见后文。

### 2.2 渲染层状态管理

关键文件：

- `src/stores/file.ts`
- `src/stores/settings.ts`
- `src/stores/editor.ts`

`file.ts` 管理多标签、当前活动标签、文件路径、脏状态、保存、打开文件、文件 watcher 和工作区路径。

`settings.ts` 管理主题、编辑模式、字号、自动保存、侧边栏、大纲、状态栏、专注模式、打字机模式、自定义 CSS 和最近文件。

`editor.ts` 管理光标位置、字数统计、滚动请求、搜索请求、标题请求和格式化请求。

### 2.3 编辑器与预览

关键文件：

- `src/components/editor/EditorContainer.vue`
- `src/components/editor/MarkdownSource.vue`
- `src/components/editor/MilkdownEditor.vue`
- `src/components/editor/WysiwygEditor.vue`
- `src/components/editor/SplitEditor.vue`
- `src/components/editor/MarkdownPreview.vue`
- `src/components/editor/SearchReplace.vue`
- `src/components/editor/TableEditor.vue`

编辑器有三种模式：

- `wysiwyg`：Milkdown 所见即所得
- `split`：CodeMirror 源码 + MarkdownPreview 预览
- `source`：CodeMirror 源码

当前源码模式功能相对完整，WYSIWYG 模式存在多项菜单命令空实现或行为不一致。

## 3. 主要问题清单

### 3.1 WYSIWYG 模式大量格式命令未实现

位置：`src/components/editor/MilkdownEditor.vue`

问题：

`handleFormatCommand()` 中多项命令直接 `break`，包括：

- `hyperlink`
- `image`
- `blockquote`
- `ordered-list`
- `unordered-list`
- `task-list`
- `insert-above`
- `insert-below`
- `footnote`
- `toc`
- `yaml-front-matter`
- `table`

影响：

应用默认编辑模式是 `wysiwyg`，但菜单和快捷键看起来可用，用户点击后没有效果，会认为功能损坏。

建议：

- 短期：在 WYSIWYG 模式下禁用未实现菜单项，或自动切到源码模式执行。
- 中期：补齐 Milkdown 对应命令，保证三种编辑模式行为一致。

优先级：高

### 3.2 WYSIWYG 注释命令可能修改错误位置

位置：`src/components/editor/MilkdownEditor.vue`

问题：

`wrapSelectedInMarkdown()` 通过 `props.modelValue.indexOf(selectedText)` 找到选中文字在 Markdown 字符串中的位置。

风险：

如果文档里有多处相同文字，实际会修改第一个匹配，而不是当前选区。

建议：

使用 ProseMirror selection 对应的位置映射，或不要在 WYSIWYG 中用字符串 `indexOf` 反查源 Markdown。

优先级：高

### 3.3 快捷键文档和实际行为冲突

位置：

- `README.md`
- `src/App.vue`
- `src/components/layout/TitleBar.vue`
- `src/components/editor/EditorContainer.vue`

问题：

`README.md` 和欢迎页写的是：

- `Ctrl+\`：切换大纲
- `Ctrl+= / Ctrl+- / Ctrl+0`：放大 / 缩小 / 重置字号

但实际代码中：

- `Ctrl+\`：触发 `clear-format`
- `Ctrl+=`：提升标题级别
- `Ctrl+-`：降低标题级别
- `Ctrl+0`：设为段落

影响：

用户按文档操作会触发完全不同功能，尤其 `Ctrl+\` 可能清除选中文本格式。

建议：

先统一快捷键设计，再同步修改 `README.md`、欢迎页、帮助弹窗和 `App.vue` 实现。

优先级：高

### 3.4 文件授权边界偏松

位置：

- `electron/preload/index.ts`
- `electron/main/ipc/file.ipc.ts`
- `src/stores/file.ts`
- `src/utils/export.ts`

问题：

preload 暴露了：

```ts
authorize: (targetPath: string) => ipcRenderer.invoke('file:grantAccess', targetPath)
```

renderer 可以主动把任意路径加入允许列表。

影响：

目前 renderer 是可信应用代码时问题不大，但如果未来引入不可信 Markdown HTML、插件、远程内容、供应链脚本或 XSS，文件读写边界会被扩大。

建议：

- 不要让 renderer 任意 grant access。
- 授权应只来自系统对话框选择结果。
- 对保存图片、导出读取资源等场景做路径限制，例如只允许文档目录及其子目录。

优先级：中高

### 3.5 导出本地图片可能读取任意本地文件

位置：`src/utils/export.ts`

问题：

导出时 `inlineLocalImages()` 会解析 Markdown 中的 `<img src>`，然后：

1. `file.authorize(absolutePath)`
2. `file.readBinary(absolutePath)`
3. 转成 data URL 嵌入导出 HTML/PDF/Word

影响：

如果文档中包含类似本地敏感文件路径，导出流程会尝试读取并内联。虽然文件作为图片可能无法正常显示，但仍可能把本地文件内容写入导出结果。

建议：

- 只内联文档目录下的相对图片。
- 禁止自动读取绝对路径，或弹窗确认。
- 限制扩展名和 MIME 类型。
- 限制文件大小。

优先级：中高

### 3.6 删除文件树节点时先关闭标签，删除失败会丢 UI 状态

位置：`src/components/sidebar/FileTree.vue`

问题：

`deleteNode()` 中先执行：

```ts
fileStore.closeTabsInPath(node.path)
await window.electronAPI?.file.delete(node.path)
```

风险：

如果 `file.delete()` 失败，标签已经关闭。对于用户来说，未保存内容或编辑上下文可能已经丢失。

建议：

先删除文件，成功后再关闭相关标签；或者保留待关闭标签快照，失败时恢复。

优先级：中

### 3.7 搜索替换存在替换语义不一致

位置：`src/components/editor/SearchReplace.vue`

问题：

非正则模式下：

- `replaceCurrent()` 是字面量替换。
- `replaceAll()` 使用 `String.replace(pattern, replaceText.value)`。

风险：

替换文本包含 `$&`、`$1` 等特殊替换语法时，全部替换会被 JS 解释，而不是按字面量插入。

建议：

非正则模式下使用 replacement function：

```ts
content.replace(pattern, () => replaceText.value)
```

优先级：中

### 3.8 清除格式对图片语法处理顺序错误

位置：`src/components/editor/MarkdownSource.vue`

问题：

`clearFormat()` 先处理链接：

```ts
.replace(/\[(.+?)\]\(.+?\)/g, '$1')
.replace(/!\[(.+?)\]\(.+?\)/g, '$1')
```

对于 `![alt](url)`，链接规则会先匹配 `[alt](url)`，结果可能留下 `!alt`。

建议：

先处理图片，再处理普通链接。

优先级：中

### 3.9 导出菜单反馈不可见

位置：`src/components/export/ExportMenu.vue`

问题：

点击导出后立即：

```ts
handleExport(type)
closeMenu()
```

组件卸载后，`exportStatus` 即使更新为“导出成功 / 失败”，用户也看不到。

建议：

- 导出菜单不要立即关闭。
- 或把导出状态放到全局 toast / 状态栏。
- 失败时展示具体错误。

优先级：中低

### 3.10 WYSIWYG 粘贴图片总是追加到文档末尾

位置：`src/components/editor/WysiwygEditor.vue`

问题：

WYSIWYG 粘贴图片后：

```ts
emit('update:modelValue', props.modelValue + mdTag)
```

影响：

图片不会插入当前光标位置，而是追加到全文末尾。长文档中体验较差。

建议：

通过 Milkdown / ProseMirror 当前 selection 插入 image node 或 Markdown 图片文本。

优先级：中低

## 4. 功能欠缺

### 4.1 测试与质量保障不足

当前项目没有：

- 单元测试
- 组件测试
- E2E 测试
- `lint` 脚本
- `format` 脚本
- CI 配置

建议优先补：

- Vitest：测试纯函数和 store 行为。
- Playwright：覆盖打开文件、编辑、保存、关闭提示、导出入口等关键流程。
- ESLint + Prettier：减少格式和潜在问题漂移。

### 4.2 保存与关闭流程不统一

窗口关闭有“保存并关闭 / 不保存关闭 / 取消”，但关闭单个标签只有 `window.confirm()`。

建议：

- 关闭单个标签也提供“保存 / 不保存 / 取消”。
- 对 `closeOtherTabs`、`closeAllTabs`、删除节点等路径统一使用同一套确认逻辑。

### 4.3 文件类型和文件大小保护不足

当前打开文件时没有明显的：

- 大文件提示
- 二进制文件检测
- 编码检测
- 只读文件处理
- 保存失败 UI 提示

建议：

- 对超过一定大小的文件提示用户。
- 非 UTF-8 或二进制内容拒绝作为 Markdown 打开。
- 保存失败时弹出明确错误。

### 4.4 状态持久化不完整

已持久化：

- 主题
- 编辑模式
- 自动保存
- 自动保存间隔
- 字号
- 自定义 CSS 路径
- 最近文件

未持久化：

- 工作区路径
- 打开的标签页
- 侧边栏开关
- 大纲开关
- 状态栏开关
- 专注模式
- 打字机模式
- 分栏宽度

建议：

增加统一的设置持久化层，避免 scattered localStorage。

### 4.5 工作区能力偏基础

当前文件树支持目录读取、展开、重命名、新建、删除、刷新，但缺少：

- 工作区搜索文件
- 文件树自动刷新
- 根目录空白处右键新建
- 拖拽移动文件
- 删除到回收站
- 删除撤销

建议：

优先补文件搜索、自动刷新和删除到回收站。

### 4.6 编辑体验仍不一致

源码模式比 WYSIWYG 更完整。

缺口包括：

- WYSIWYG 下搜索需要强制切到源码。
- WYSIWYG 下多项格式命令未实现。
- WYSIWYG 下图片插入位置不正确。
- WYSIWYG 对 Mermaid / 数学公式的所见即所得支持有限。
- 搜索没有高亮全部匹配。

建议：

把编辑命令抽象成统一能力层，分别适配 CodeMirror 和 Milkdown。

### 4.7 导出体验可增强

当前支持 PDF / HTML / Word，但缺少：

- 导出进度
- 导出完成后打开位置
- 失败原因展示
- PDF 页眉页脚配置
- 图片缺失列表
- 大文档导出保护
- HTML 资源策略配置

建议：

先加全局导出状态和错误提示，再补导出选项。

### 4.8 发布维护能力不足

当前 `package.json` 中：

```json
"build": "vite build && electron-builder"
```

问题：

- build 前没有强制 type-check。
- 没有测试。
- 检查更新硬编码仓库 `anxin233/md-editor`。
- 版本比较不处理 prerelease。

建议：

- 增加 `check` 脚本：`type-check + lint + test`。
- build 前至少执行 type-check。
- 更新检查逻辑抽离配置。
