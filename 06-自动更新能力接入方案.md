# 06-自动更新能力接入方案

关联问题：当前项目的“检查更新”只能打开 GitHub Release 页面，用户需要手动下载安装包，无法在应用内自动下载和安装更新。

## 1. 当前现状

当前项目已有更新检查入口，主要逻辑在：

- `src/components/layout/TitleBar.vue`

现有流程是：

1. renderer 通过 `fetch()` 请求 GitHub Releases API。
2. 对比当前版本和最新 Release tag。
3. 如果发现新版本，展示“前往下载”按钮。
4. 用户点击后打开 GitHub Release 页面。
5. 用户手动下载安装包并安装。

这属于“检查更新 + 手动下载”，不是自动更新。

当前项目还没有：

- `electron-updater`
- `autoUpdater`
- `latest.yml` 自动更新元数据配置
- 自动下载进度
- 下载完成后重启安装入口

## 2. 目标

接入 Electron 标准自动更新能力，使应用支持：

1. 应用内检查更新。
2. 发现新版本后自动下载安装包。
3. 展示下载进度。
4. 下载完成后提示用户“立即重启安装”。
5. 用户确认后自动退出并安装。
6. 后续发布版本时，通过 GitHub Release 自动分发更新。

注意：已经安装的旧版本如果没有内置自动更新能力，无法自动升级到自动更新版。需要用户最后手动安装一次带自动更新的新版本，之后才可自动更新。

## 3. 推荐技术方案

推荐使用：

```text
electron-updater + electron-builder publish + GitHub Releases
```

原因：

- 项目已经使用 `electron-builder`。
- `electron-updater` 与 `electron-builder` 集成度最高。
- Windows NSIS 安装包支持成熟。
- GitHub Releases 可以作为更新源。

自动更新主要面向 Windows `nsis` 安装版。`portable` 版本通常不适合作为自动更新目标。

## 4. 依赖调整

新增运行时依赖：

```bash
npm install electron-updater
```

不要放到 `devDependencies`，因为主进程运行时需要使用它。

## 5. package.json 配置

### 5.1 配置发布源

在 `package.json` 的 `build` 字段中增加：

```json
"publish": [
  {
    "provider": "github",
    "owner": "anxin233",
    "repo": "md-editor",
    "releaseType": "release"
  }
]
```

如果仓库后续改名，应同步修改 `owner` 和 `repo`。

### 5.2 增加发布脚本

建议新增：

```json
"release:win": "vite build && electron-builder --win --publish always"
```

可选增加：

```json
"release:win:draft": "vite build && electron-builder --win --publish onTagOrDraft"
```

推荐先用 `--publish always` 明确上传产物，避免漏传更新元数据。

## 6. GitHub Release 要求

自动更新依赖 GitHub Release 中的安装包和更新元数据。

Windows NSIS 发布后 Release 中至少应包含：

- `MDEditor-Setup-<version>-win-x64.exe`
- `MDEditor-Setup-<version>-win-x64.exe.blockmap`
- `latest.yml`

其中 `latest.yml` 是自动更新的关键文件。如果只有 `.exe` 安装包但没有 `latest.yml`，客户端无法正常自动发现和下载更新。

## 7. 主进程设计

### 7.1 新增 update IPC 文件

新增：

```text
electron/main/ipc/update.ipc.ts
```

职责：

1. 初始化 `autoUpdater`。
2. 接收 renderer 的检查更新请求。
3. 将更新状态推送给 renderer。
4. 下载完成后执行重启安装。

### 7.2 状态类型

建议定义状态：

```ts
type UpdateStatus =
  | { state: 'idle' }
  | { state: 'checking' }
  | { state: 'latest' }
  | { state: 'available'; version?: string }
  | { state: 'downloading'; percent: number }
  | { state: 'downloaded'; version?: string }
  | { state: 'error'; message: string }
  | { state: 'disabled'; message: string }
```

### 7.3 主进程核心逻辑

建议实现：

```ts
import { app, BrowserWindow, ipcMain } from 'electron'
import { autoUpdater } from 'electron-updater'

let mainWindow: BrowserWindow | null = null

export function registerUpdateIpcHandlers(win: BrowserWindow) {
  mainWindow = win

  autoUpdater.autoDownload = true
  autoUpdater.autoInstallOnAppQuit = true

  autoUpdater.on('checking-for-update', () => {
    sendUpdateStatus({ state: 'checking' })
  })

  autoUpdater.on('update-available', info => {
    sendUpdateStatus({ state: 'available', version: info.version })
  })

  autoUpdater.on('update-not-available', () => {
    sendUpdateStatus({ state: 'latest' })
  })

  autoUpdater.on('download-progress', progress => {
    sendUpdateStatus({
      state: 'downloading',
      percent: progress.percent,
    })
  })

  autoUpdater.on('update-downloaded', info => {
    sendUpdateStatus({ state: 'downloaded', version: info.version })
  })

  autoUpdater.on('error', error => {
    sendUpdateStatus({
      state: 'error',
      message: error.message,
    })
  })

  ipcMain.handle('update:check', async () => {
    if (!app.isPackaged) {
      return {
        state: 'disabled',
        message: '开发环境不检查自动更新',
      }
    }

    await autoUpdater.checkForUpdates()
    return { state: 'checking' }
  })

  ipcMain.handle('update:install', () => {
    autoUpdater.quitAndInstall()
  })
}

function sendUpdateStatus(payload: unknown) {
  mainWindow?.webContents.send('update:status', payload)
}
```

### 7.4 注册位置

在 `electron/main/index.ts` 中：

```ts
import { registerUpdateIpcHandlers } from './ipc/update.ipc'
```

创建窗口后注册：

```ts
createWindow()

if (mainWindow) {
  registerUpdateIpcHandlers(mainWindow)
}
```

或在 `createWindow()` 内部 `mainWindow` 创建完成后直接注册。

注意不要重复注册 `ipcMain.handle('update:check')`，避免多窗口或重复调用时出现 handler 重复问题。

## 8. preload API 设计

在 `electron/preload/index.ts` 中增加：

```ts
update: {
  check: () => Promise<UpdateStatus>
  install: () => Promise<void>
  onStatus: (callback: (status: UpdateStatus) => void) => () => void
}
```

实现示例：

```ts
update: {
  check: () => ipcRenderer.invoke('update:check'),
  install: () => ipcRenderer.invoke('update:install'),
  onStatus: (callback) => {
    const handler = (_event: Electron.IpcRendererEvent, data: UpdateStatus) => callback(data)
    ipcRenderer.on('update:status', handler)
    return () => ipcRenderer.removeListener('update:status', handler)
  },
}
```

## 9. 前端 UI 改造

### 9.1 替换当前 GitHub API 检查

当前 `TitleBar.vue` 中直接请求：

```ts
fetch(`https://api.github.com/repos/${GITHUB_REPO}/releases/latest`)
```

应替换为：

```ts
await window.electronAPI.update.check()
```

前端不再直接访问 GitHub API，不再自己比较版本。

版本判断、下载、安装都交给 `electron-updater`。

### 9.2 更新状态

建议 UI 状态改为：

```ts
const updateState = ref<
  'idle'
  | 'checking'
  | 'latest'
  | 'available'
  | 'downloading'
  | 'downloaded'
  | 'error'
  | 'disabled'
>('idle')
```

新增：

```ts
const updateProgress = ref(0)
const updateVersion = ref('')
const updateError = ref('')
```

### 9.3 监听更新状态

在 `onMounted()` 中：

```ts
let updateCleanup: (() => void) | null = null

onMounted(() => {
  updateCleanup = window.electronAPI?.update.onStatus((status) => {
    updateState.value = status.state

    if ('version' in status && status.version) {
      updateVersion.value = status.version
    }

    if (status.state === 'downloading') {
      updateProgress.value = Math.round(status.percent)
    }

    if (status.state === 'error') {
      updateError.value = status.message
    }
  }) || null
})

onUnmounted(() => {
  updateCleanup?.()
})
```

### 9.4 检查更新按钮

```ts
async function checkUpdates() {
  closeMenu()
  showUpdateDialog.value = true
  updateState.value = 'checking'
  updateError.value = ''
  updateProgress.value = 0

  const result = await window.electronAPI?.update.check()
  if (result?.state === 'disabled') {
    updateState.value = 'disabled'
    updateError.value = result.message
  }
}
```

### 9.5 安装按钮

下载完成后展示：

```text
新版本已下载，是否立即重启安装？
```

按钮调用：

```ts
function installUpdate() {
  window.electronAPI?.update.install()
}
```

## 10. 推荐用户体验

### 10.1 手动检查

保留当前“帮助 / 检查更新”入口：

1. 用户点击检查更新。
2. 弹窗展示检查状态。
3. 有新版本时自动下载。
4. 下载完成后显示“立即重启安装”。

### 10.2 启动自动检查

可以在应用启动后延迟检查一次：

```ts
setTimeout(() => {
  window.electronAPI?.update.check()
}, 10_000)
```

建议第一版先只做“用户手动点击检查更新”，稳定后再加启动自动检查。

### 10.3 避免干扰写作

下载完成后不要强制重启，应该让用户选择：

- 立即重启安装
- 稍后安装

`autoInstallOnAppQuit = true` 可以保证用户退出应用时自动安装。

## 11. 发布流程

以后发布新版本时：

### 11.1 修改版本号

修改 `package.json`：

```json
"version": "1.0.2"
```

### 11.2 本地检查

```bash
npm run type-check
npm run build
```

注意：如果 `npm run build` 会直接执行 `electron-builder`，也可以先执行：

```bash
npx vite build
```

### 11.3 设置 GitHub Token

PowerShell：

```powershell
$env:GH_TOKEN="你的 GitHub token"
```

Token 需要有创建 Release 和上传 Release assets 的权限。

### 11.4 发布 Windows 安装版

```bash
npm run release:win
```

发布完成后检查 GitHub Release，确认包含：

```text
MDEditor-Setup-<version>-win-x64.exe
MDEditor-Setup-<version>-win-x64.exe.blockmap
latest.yml
```

## 12. 版本号要求

`electron-updater` 使用语义化版本比较。

建议始终使用：

```text
1.0.2
1.0.3
1.1.0
```

Git tag 推荐使用：

```text
v1.0.2
```

避免随意使用：

```text
release-1
版本1
1.0
```

## 13. 代码签名说明

Windows 自动更新在未签名情况下也能运行，但体验可能受影响：

- SmartScreen 可能提示风险。
- 企业环境可能拦截未知发布者安装包。
- 用户信任度较低。

建议短期先支持未签名自动更新，长期再补代码签名证书。

macOS 自动更新通常更依赖签名和 notarization，本项目如果优先 Windows，可以先不处理 macOS 自动更新。

## 14. 安全注意事项

1. 不要让 renderer 自己下载安装包并执行。
2. 下载和安装必须由 `electron-updater` 在主进程中处理。
3. 更新源固定为项目 GitHub Release。
4. 不要从 renderer 传入任意 URL 给主进程更新。
5. 保持 `openExternal` 只允许 `https://`。
6. 发布 Release 时不要手动漏传 `latest.yml`。

## 15. 验收标准

完成后应满足：

1. `package.json` 依赖中包含 `electron-updater`。
2. `build.publish` 已配置 GitHub provider。
3. 主进程注册 `update:check`、`update:install`。
4. preload 暴露 `electronAPI.update.check/install/onStatus`。
5. `TitleBar.vue` 不再直接请求 GitHub Releases API 作为主要更新逻辑。
6. 点击“检查更新”能触发 `autoUpdater.checkForUpdates()`。
7. 有新版本时能自动下载。
8. 下载中能显示进度。
9. 下载完成后能点击“立即重启安装”。
10. GitHub Release 中包含 `latest.yml` 和 `.blockmap`。
11. `npm run type-check` 通过。
12. `npx vite build` 通过。

## 16. 结论

当前项目可以接入自动更新，且最适合的路线是 `electron-updater + electron-builder + GitHub Releases`。

接入后，用户只需要手动安装一次包含自动更新能力的新版本。后续版本发布到 GitHub Release 后，应用即可在内部检查、下载，并提示用户重启安装，不再需要每次去 GitHub 页面手动下载安装包。
