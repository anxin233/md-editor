# 自动更新与打包版本号 — 实现说明与验证清单

本文档汇总当前仓库中与 **Electron 自动更新（electron-updater）**、**打包前时间戳版本号** 相关的实现，并给出按序号执行的验证步骤，便于逐项核对。

---

## 一、实现概要（按序号）

1. **运行时依赖** `electron-updater` 已写入 `package.json` 的 `dependencies`（非 `devDependencies`），随应用安装进产物。
2. **构建配置** `package.json` → `build.publish` 已配置 GitHub：`provider: github`，`owner: anxin233`，`repo: md-editor`，`releaseType: release`，供 builder / updater 解析更新源。
3. **主进程** 新增 `electron/main/ipc/update.ipc.ts`：初始化 `autoUpdater`（`autoDownload`、`autoInstallOnAppQuit`），将状态通过通道 `update:status` 推送到渲染进程；注册 IPC：`update:check`（`!app.isPackaged` 时返回 `disabled`）、`update:install`（`quitAndInstall()`）；`registerUpdateIpcHandlers()` 仅注册一次。
4. **主进程入口** `electron/main/index.ts`：在 `app.whenReady` 内调用 `registerUpdateIpcHandlers()`；在 `createWindow` 末尾 `attachUpdateTargetWindow(mainWindow)`，窗口 `closed` 时 `attachUpdateTargetWindow(null)`。
5. **Vite 主进程打包** `vite.config.ts`：`rollupOptions.external` 包含 `electron-updater`，运行时从 `node_modules` 加载，避免打进单 bundle 引发路径问题。
6. **Preload** `electron/preload/index.ts`：导出类型 `UpdateStatus`、`UpdateCheckInvokeResult`；暴露 `electronAPI.update.check`、`install`、`onStatus`（监听 `update:status`）。
7. **前端** `src/components/layout/TitleBar.vue`：「检查更新」改为调用 `electronAPI.update.check()` 与 `onStatus`，不再使用 `fetch` 访问 GitHub Releases API；UI 覆盖 `checking` / `latest` / `available` / `downloading`（进度条）/ `downloaded`（立即重启安装 / 稍后）/ `error` / `disabled`；保留「查看所有版本」外链。
8. **类型** `src/electron.d.ts` 仍从 preload 的 `ElectronAPI` 继承全局 `window.electronAPI`；`TitleBar` 通过 `@preload/index` 引入 `UpdateStatus`。
9. **npm 脚本** `release:win`、`release:win:draft` 在 `vite build` 前执行版本 bump，并以 `--publish always` / `onTagOrDraft` 上传（需配置 `GH_TOKEN` 等）。
10. **打包前版本号** 新增 `scripts/bump-version-pack.ts`：将版本写为 **`年.MMDD.HHMM`**（第 2 段 = 月×100+日，第 3 段 = 时×100+分），同步更新 `package.json` 与 `package-lock.json` 根 `version`。
11. **触发 bump 的命令**（均在 `vite build` 之前执行 bump）：`build`、`build:win`、`build:mac`、`build:linux`、`build:portable`、`release:win`、`release:win:draft`。**不**影响 `dev`、`preview`、`type-check`。
12. **独立命令** `npm run version:bump-pack`：仅执行版本 bump，不构建。

## 二、版本号规则（便于人工核对）

| 段 | 含义 | 示例（2026 年 4 月 28 日 15:30） |
|----|------|----------------------------------|
| 1 | 四位年份 | `2026` |
| 2 | 月×100 + 日 | `428` |
| 3 | 时×100 + 分 | `1530` |

示例完整版本：`2026.428.1530`。

## 三、验证清单（建议按序号执行）

1. 执行 `npm run type-check`，应无报错。
2. 执行 `npm run dev`，打开「帮助 → 检查更新」：应出现 **开发环境不检查自动更新** 一类提示（`disabled`），属预期。
3. 执行 `npm run version:bump-pack`，控制台打印 `[bump-version-pack] <版本>`，且 `package.json` / `package-lock.json` 根 `version` 变为当前时间对应的 `年.MMDD.HHMM`；验证后按需 `git checkout -- package.json package-lock.json` 还原（若不想提交 bump）。
4. 执行 `npm run build:win`（或完整 `build`）：构建开始前应再次 bump；产物目录 `release/<version>/` 下安装包与 `latest.yml` 等中的版本应与 bump 后一致。
5. 安装 **NSIS 安装版**（非 portable），在「关于」中核对版本号与 `package.json` 中版本一致。
6. 在已发布 **更高版本** 的 GitHub Release 上保证存在 **`latest.yml`、`.blockmap`、安装包**（发布脚本或手动上传完整），在安装版内点「检查更新」：应能发现更新、下载并出现进度；下载完成后可测「立即重启安装」。
7. 确认 **未** 将仅含 `.exe`、缺少 `latest.yml` 的 Release 当作更新源（否则自动更新可能失败）。
8. 核对 `vite.config.ts` 中主进程 `external` 含 `electron-updater`。
9. 核对 `electron/main/index.ts` 中仅调用一次 `registerUpdateIpcHandlers`，且窗口创建后已 `attachUpdateTargetWindow`。
