# 04-快捷键体系统一与文档同步方案

关联审查项：`01-项目审查报告.md` 第 3.3 节  
问题标题：快捷键文档和实际行为冲突

## 1. 问题背景

当前项目中，快捷键定义和快捷键说明分散在多个文件里：

- `src/App.vue`：真正处理键盘事件。
- `src/components/layout/TitleBar.vue`：帮助弹窗里的快捷键说明，以及菜单项右侧的快捷键文案。
- `src/components/editor/EditorContainer.vue`：欢迎页快捷键说明。
- `src/components/layout/StatusBar.vue`：状态栏 tooltip。
- `README.md`：用户文档。

这些位置没有共享同一份数据源，导致实际行为和说明不一致。

当前已确认的冲突：

1. `README.md` 和欢迎页写 `Ctrl+\` 是“切换大纲”，但实际 `App.vue` 中 `Ctrl+\` 触发 `clear-format`。
2. `README.md`、欢迎页、状态栏 tooltip 写 `Ctrl+= / Ctrl+- / Ctrl+0` 是“放大 / 缩小 / 重置字号”，但实际 `App.vue` 中它们是“提升标题级别 / 降低标题级别 / 段落”。
3. 实际大纲快捷键是 `Ctrl+Shift+|`，但菜单里“大纲面板”没有展示快捷键，README 和欢迎页也没有正确描述。
4. 视图菜单里存在“放大字号 / 缩小字号 / 重置字号”菜单项，但当前没有对应键盘快捷键。

这是一个体系问题，不应只改 README 或某几个文案，否则后续新增快捷键还会再次漂移。

## 2. 解决目标

本项目还在开发阶段，因此不需要兼容错误文档或旧行为。目标是一次性建立长期机制：

1. 快捷键只允许在一个地方声明。
2. 键盘监听、菜单展示、帮助弹窗、欢迎页、README 都从同一份声明派生。
3. 新增或修改快捷键时，必须同时影响运行时行为和所有展示文案。
4. 明确哪些功能有快捷键，哪些功能只有菜单入口，不再用错误 tooltip 暗示存在快捷键。
5. 为后续测试提供可枚举的快捷键清单。

### 3.2 调整的大纲快捷键

当前代码用：

```ts
if (ctrl && e.shiftKey && e.key === '|') {
  settingsStore.toggleToc()
}
```

建议最终改为基于物理键位：

```ts
Ctrl+Shift+Backslash
```

展示文案统一为：

```text
Ctrl+Shift+\
```

原因：

- 避免依赖 `e.key === '|'`，不同键盘布局和输入法下不稳定。
- 避免和 `Ctrl+\` 清除样式冲突。
- 大纲属于视图开关，使用 `Shift` 作为增强组合键合理。

### 3.3 字号功能暂不分配快捷键

当前 `view:font-inc`、`view:font-dec`、`view:font-reset` 只有菜单和状态栏入口，没有真实键盘监听。

建议本阶段不要给字号功能分配键盘快捷键，理由：

- `Ctrl+= / Ctrl+- / Ctrl+0` 已被段落/标题编辑占用。
- Markdown 编辑器里标题和段落快捷键比字号更高频。
- 如果强行给字号补快捷键，会继续增加冲突。

因此最终规则是：

- 字号调整保留菜单入口和状态栏显示。
- 所有文档、tooltip、欢迎页不得再写 `Ctrl+= / Ctrl+- / Ctrl+0` 用于字号。
- 后续如果确实需要字号快捷键，必须先在快捷键注册表中新增，并做冲突检测。

## 4. 架构方案：建立快捷键注册表

新增文件：

```text
src/utils/shortcutRegistry.ts
```

它是项目内快捷键的唯一事实来源。

### 4.1 数据结构

建议定义：

```ts
export type ShortcutScope = 'global' | 'editor' | 'view' | 'file'

export interface ShortcutDefinition {
  id: string
  label: string
  keys: string[]
  scope: ShortcutScope
  command: AppCommandDetail | { type: 'internal'; action: string }
  visibleInHelp?: boolean
  visibleInWelcome?: boolean
  preventDefault?: boolean
}
```

说明：

- `id`：稳定唯一 ID，便于测试和引用。
- `label`：用户可见说明。
- `keys`：统一展示格式，例如 `Ctrl+Shift+\`。
- `scope`：后续支持不同区域快捷键。
- `command`：与 `AppCommandDetail` 对齐，避免键盘处理和菜单命令分裂。
- `visibleInHelp`：是否进入帮助弹窗。
- `visibleInWelcome`：是否进入欢迎页。
- `preventDefault`：是否阻止浏览器 / Electron 默认行为。

### 4.2 示例声明

```ts
export const SHORTCUTS: ShortcutDefinition[] = [
  {
    id: 'file.new',
    label: '新建文件',
    keys: ['Ctrl+N'],
    scope: 'file',
    command: { type: 'file:new' },
    visibleInHelp: true,
    visibleInWelcome: true,
    preventDefault: true,
  },
  {
    id: 'view.toggle-toc',
    label: '显示 / 隐藏大纲',
    keys: ['Ctrl+Shift+\\'],
    scope: 'view',
    command: { type: 'view:toggle-toc' },
    visibleInHelp: true,
    visibleInWelcome: true,
    preventDefault: true,
  },
  {
    id: 'format.clear',
    label: '清除样式',
    keys: ['Ctrl+\\'],
    scope: 'editor',
    command: { type: 'format', command: 'clear-format' },
    visibleInHelp: true,
    visibleInWelcome: false,
    preventDefault: true,
  },
]
```

## 5. 键盘匹配方案

不要继续在 `App.vue` 中写大量 `if (ctrl && e.key === ...)`。

应新增标准化匹配函数：

```ts
export function normalizeKeyboardEvent(event: KeyboardEvent): string {
  const parts: string[] = []
  if (event.ctrlKey || event.metaKey) parts.push('Ctrl')
  if (event.shiftKey) parts.push('Shift')
  if (event.altKey) parts.push('Alt')

  const key = normalizeKey(event)
  parts.push(key)
  return parts.join('+')
}
```

`normalizeKey()` 需要特别处理：

- `event.code === 'Backslash'` → `\`
- `event.code === 'BracketLeft'` → `[`
- `event.code === 'BracketRight'` → `]`
- `event.code === 'Backquote'` → `` ` ``
- `event.code === 'Digit5'` + Shift → `5`
- `event.key` 大小写统一

原因：

- `e.key` 受键盘布局、Shift、输入法影响。
- `e.code` 更适合快捷键物理键位判断。
- 展示给用户的仍然是 `Ctrl+Shift+\` 这类可读格式。

## 6. App.vue 改造方案

当前：

```ts
function setupKeyboardShortcuts() {
  keydownHandler = async (e: KeyboardEvent) => {
    const ctrl = e.ctrlKey || e.metaKey

    if (ctrl && e.key === 'n') {
      // ...
    }

    // 大量 if 分支
  }
}
```

建议改为：

```ts
function setupKeyboardShortcuts() {
  keydownHandler = async (event: KeyboardEvent) => {
    const shortcut = findShortcutByEvent(event)
    if (!shortcut) return

    if (shortcut.preventDefault !== false) {
      event.preventDefault()
    }

    await executeShortcut(shortcut)
  }

  window.addEventListener('keydown', keydownHandler)
}
```

执行逻辑：

```ts
async function executeShortcut(shortcut: ShortcutDefinition) {
  if ('type' in shortcut.command && shortcut.command.type !== 'internal') {
    await handleAppCommand(shortcut.command)
    return
  }

  // 处理少量内部动作，比如 Ctrl+Tab 由主进程发送事件，不走普通 keydown。
}
```

这样可以保证快捷键行为和 `AppCommandDetail` 使用同一套命令。

## 7. 菜单和帮助弹窗改造方案

### 7.1 TitleBar 帮助弹窗

当前 `shortcutGroups` 是硬编码数组，应改为从注册表派生：

```ts
const shortcutGroups = computed(() => {
  return SHORTCUTS
    .filter(item => item.visibleInHelp)
    .map(item => [formatShortcutKeys(item.keys), item.label])
})
```

这样帮助弹窗不再手写快捷键。

### 7.2 菜单右侧快捷键

新增 helper：

```ts
function shortcutLabel(command: AppCommandDetail): string {
  return findShortcutByCommand(command)?.keys.join(' / ') || ''
}
```

菜单中不再写死：

```vue
<span class="shortcut">Ctrl+E</span>
```

而是：

```vue
<span class="shortcut">{{ shortcutLabel({ type: 'view:set-mode', mode: 'wysiwyg' }) }}</span>
```

注意：`view:set-mode` 三个菜单项共用 `Ctrl+E` 循环切换模式，这种情况可以保留同一个显示值。

### 7.3 大纲菜单项

当前菜单大纲项没有快捷键展示：

```vue
<button class="dropdown-item" @click="runCommand({ type: 'view:toggle-toc' })">
  <span>大纲面板</span>
```

应显示：

```text
Ctrl+Shift+\
```

这两个都应移除硬编码。

```ts
const welcomeShortcuts = computed(() =>
```

## 10. README 同步方案

`README.md` 不应长期手写快捷键表。

推荐新增脚本：

```text
scripts/update-readme-shortcuts.ts
```

脚本做两件事：

1. 读取 `src/utils/shortcutRegistry.ts` 的快捷键声明。
2. 替换 README 中固定标记之间的内容。

README 使用标记：

```md
<!-- shortcuts:start -->
| 快捷键 | 作用 |
|--------|------|
| Ctrl+N | 新建文件 |
<!-- shortcuts:end -->
```

以后修改快捷键后执行：

```bash
npm run docs:shortcuts
```

`package.json` 增加：

```json
{
  "scripts": {
    "docs:shortcuts": "tsx scripts/update-readme-shortcuts.ts"
  }
}
```

如果暂时不想引入 `tsx`，也可以用纯 Node + TypeScript 编译产物，但从长期维护角度，脚本生成是最稳的。

## 11. 冲突检测

新增函数：

```ts
export function validateShortcutRegistry(shortcuts: ShortcutDefinition[]) {
  const seen = new Map<string, string>()

  for (const shortcut of shortcuts) {
    for (const key of shortcut.keys) {
      const normalized = key.toLowerCase()
      const existing = seen.get(normalized)
      if (existing) {
        throw new Error(`快捷键冲突：${key} 同时用于 ${existing} 和 ${shortcut.id}`)
      }
      seen.set(normalized, shortcut.id)
    }
  }
}
```

在开发环境启动时调用：

```ts
if (import.meta.env.DEV) {
  validateShortcutRegistry(SHORTCUTS)
}
```

也建议后续写单元测试：

```ts
expect(() => validateShortcutRegistry(SHORTCUTS)).not.toThrow()
```

## 13. 分阶段实施计划

### 第一阶段：建立唯一事实来源

1. 新增 `shortcutRegistry.ts`。
2. 把当前 `App.vue` 中所有快捷键迁移到注册表。
3. 实现 `findShortcutByEvent()` 和 `executeShortcut()`。
4. 保证现有快捷键行为不变，除了大纲快捷键从 `Ctrl+Shift+|` 规范为 `Ctrl+Shift+\`。

### 第二阶段：移除硬编码文案

1. `TitleBar.vue` 帮助弹窗从注册表生成。
2. 菜单右侧快捷键从注册表查询。
3. `EditorContainer.vue` 欢迎页从注册表生成。
4. `StatusBar.vue` 移除错误字号快捷键 tooltip。

### 第三阶段：README 自动同步

1. 为 README 快捷键表添加 `shortcuts:start/end` 标记。
2. 新增 `docs:shortcuts` 脚本。
3. 执行脚本生成最新快捷键表。
4. 在开发流程中约定：改快捷键必须运行 `npm run docs:shortcuts`。

### 第四阶段：校验和测试

1. 增加快捷键冲突检测。
2. 增加注册表单元测试。
3. 手动验证关键快捷键。
4. 跑 `npm run type-check`。

## 14. 验收标准

完成后必须满足：

1. `Ctrl+\` 在所有地方都只表示“清除样式”。
2. `Ctrl+Shift+\` 在所有地方都只表示“显示 / 隐藏大纲”。
3. `Ctrl+= / Ctrl+- / Ctrl+0` 在所有地方都只表示“提升标题级别 / 降低标题级别 / 段落”。
4. 字号功能不再显示任何不存在的快捷键。
5. `App.vue` 不再维护散落的快捷键 `if` 分支。
6. 帮助弹窗、菜单快捷键、欢迎页、README 都来自同一份快捷键注册表。
7. 快捷键注册表冲突检测通过。
8. `npm run type-check` 通过。

## 15. 结论

3.3 的根因不是某一处文档写错，而是快捷键缺少统一数据源。

一劳永逸的解决方式是建立 `shortcutRegistry.ts`，让运行时处理、菜单展示、欢迎页、帮助弹窗和 README 全部由它派生。项目仍在开发阶段，应直接替换现有分散写法，不需要保留错误快捷键或做兼容层。
